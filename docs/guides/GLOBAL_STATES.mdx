# Global States<br/><br/>

The state styles for each component are declared inside their tokens. For every component we get the following tokens:

- **_inactive_** (used for the **_default state_** of the component)
- **_hover_** (used for the **_Hover_** state)
- **_focused_** (used for **_Focus_** state)
- **_active_** (used for the **_Pressed_** state)
- **_disabled_** (used for **_Disabled_** state)

The component tokens for these states point to the semantic tokens, so basically the states are centralised based on the chosen theme.

## Hover

When hovering, the component's corresponding token returns the **_light variant of the chosen systemic color_** for the background.

## Focused

When focused, the component's corresponding token returns a **_border declared in the semantic tokens_**.

## Pressed

When pressed, the component's corresponding token returns the **_dark variant of the chosen systemic color_** for the background.

## Populated

No rule is defined for this yet.

## Disabled

When disabled, the component's corresponding token returns the **_disabled-state semantic token_**.
