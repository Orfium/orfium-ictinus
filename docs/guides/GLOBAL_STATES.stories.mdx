# Global States<br/><br/>

In order to provide a consistent feel among apps that use Ictinus, we have common styling rules on the interactions of the user with some components

## Hover

When hovering, the background color is darkened by one step in shade ( +50 ).


- If no background color exists, transparent or white, then on hover we use lightGrey-50
- If it goes out of bounds, it's lightened by one step ( -50 )


## Focused

When focused, the border is set to Magenta-500 with width the current width of the component + 1.

All Text components are excluded from this rule, as the focused and active rules result in the same state.

## Pressed

When pressed, the background color is darkened by two steps in shade ( +100 ).


- If no background color exists, transparent or white, then on pressed we use lightGrey-100
- If it goes out of bounds, it's lightened by two steps ( -100 )

## Populated

No Global rule is defined for this yet.


## Disabled

Apply opacity 50% to whatever style each component has.
