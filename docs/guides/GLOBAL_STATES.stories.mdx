# Global States<br/><br/>

The state styles for each component are declared inside their tokens. For every component we get the following tokens:

- ***inactive*** (used for the ***default state*** of the component)
- ***hover*** (used for the ***Hover*** state)
- ***focused*** (used for ***Focus*** state)
- ***active*** (used for the ***Pressed*** state)
- ***disabled*** (used for ***Disabled*** state)

The component tokens for these states point to the semantic tokens, so basically the states are centralised based on the chosen theme.

## Hover

When hovering, the component's corresponding token returns the ***light variant of the chosen systemic color*** for the background.

## Focused

When focused, the component's corresponding token returns a ***border declared in the semantic tokens***.

## Pressed

When pressed, the component's corresponding token returns the ***dark variant of the chosen systemic color*** for the background.

## Populated

No rule is defined for this yet.

## Disabled

When disabled, the component's corresponding token returns the ***disabled-state semantic token***.
